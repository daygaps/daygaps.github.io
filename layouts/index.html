How it works

Less to maintain. More to do.

  • Tasks find their own order. Inside each chunk, tasks sort themselves by project and area. You write what needs doing; the order takes care of itself.
  • One way to mark a task: done. A checkbox is the only knob a task needs.
  • Projects have an end. A project is a thing with a finish line — a paper, a renovation, a semester's service. Areas hold the ongoing work.
  • Completed work stays visible. You can see what you got done today, this week, all the way back to when the project started.
  • Subtasks live with their parents. A task with parts shows its parts — open, in context, nothing hidden.
